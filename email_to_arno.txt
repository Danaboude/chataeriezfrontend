### üöÄ Technical Overview & Architecture
This project was built to demonstrate a scalable real-time messaging architecture.

**Frontend:** Angular (latest version)
- **Responsive UI**: Built with Tailwind CSS, optimized for both desktop and mobile. I've implemented a WhatsApp-style floating chat bar for mobile.
- **State Management**: Uses RxJS for handling real-time data streams and Socket.IO for the connection bridge.
- **Local Storage**: For this demo, message history is stored locally in your browser's `localStorage`. In a production environment, we would integrate a persistent database (like MongoDB or PostgreSQL).

**Backend:** Node.js / Express
- **Messaging Protocol**: Uses MQTT (via RabbitMQ) for reliable and scalable message delivery. 
- **Broker**: I used **CloudAMQP** (RabbitMQ) as the managed messaging broker.
- **Bridge**: The backend acts as a bridge between Socket.IO (frontend) and MQTT (broker), ensuring low-latency communication.

**Notifications & Synchronization:**
- **Smart Notifications**: I've implemented a Service Worker to handle background notifications. You will receive notifications even if the app is closed or if you are chatting in a different room.
- **Unread Counters**: Real-time red badges appear in the sidebar to alert you of background messages.
- **Sync Logic**: The backend uses global broadcasting. This ensures that Alice receives Bob's private messages instantly, even if she is currently focused on the "General" group chat.
